1. проверяем версию Git
git --version
git --help (вызывает в помощь список команд  Git)

2. переходим в место создания проекта, например  на диск D:
cd D: (cd - chenge directory)
cd .. - переход на один уровень вверх
cd ../.. - переход на два уровня вверх

3. создаем папку 
mkdir и название папки (где mkdir - make directory)
переходим в вновь созданую папку
ls - просмотр содержимого папки директории(пусто)

4. инициалицируем проект для git
git init (создается скрытый файл инициализации .git)
для просмотра и скрытого содержимого есть команда
ls -a (полный просмотр)

5. начинаем работать с данной директорией
touch <имя> - создание файла <имя>
echo 'текст' > <имя файла> - пишем текст в файл
cat - просмотр содержимого файла
rm <имя> - удалить файл <имя>
rm -R <имя> - удалить папку <имя>

git status
git add <имя>
git add .
git commit -m 'пояснительный текст'

git branch - просмотр веток
git branch <имя ветки> - создание ветки
git branch -D <имя ветки> - удаление ветки
git checkout <имя ветки> - переход на ветку
git checkout -b <имя ветки> - создаем и сразу переходим на ветку
git merge <имя ветки> - воссоединяем ветки

6. заходим в удаленный репозиторий например GitHub
и создаем новый репозиторий в нем.
	+ new repository
		-create repository
Выполняем настройки по связыванию удаленного и локального репозиториев
	git config --global user.name Имя
	git config --global user.email почту
	git branch -M main основная ветка
git remote add origin https://________.git добавляем удаленный к локальному
git push -u origin  main заливаем на удаленный
 в дальнейшем для заливки и забора изменений используем
git pull
git push
