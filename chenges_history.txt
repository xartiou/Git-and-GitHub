Git никогда не удаляет данные  с репозиторя  - только добавляет
История хранит удаленные объекты
Каждый шаг хранит данные о всем проекте
Изменение истории вызывает проблемы

в терминале:
	просмотр истории изменений   - git log
        компактный просмотр истории  - git log --oneline
        просмотр истории действий    - git reflog
        сброс текущего состояния историй - git reset
        отмена последствий коммита   - git revert

в Kraken:
	- revert commit (отменить действия коммита)
	- reset master:
		-soft(с сохранением файлов без удаления)
		-mixed(сохранение текущих вайлов и сброс с индекса)
		-hard(сброс всех изменений всех файлов)
